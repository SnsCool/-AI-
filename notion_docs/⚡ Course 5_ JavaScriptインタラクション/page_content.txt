ゴール: JavaScriptで「ページに動きと反応」を加える力を身につける。イベント、DOM操作、条件分岐、データ取得の基礎。
Lesson 1: JavaScriptの世界へようこそ
Q1
問題: HTML＋CSSだけで作ったページのボタンを押すと、何が起きるでしょう？
正解: A. 見た目は整っているが、押しても何も起きない
解説: HTML＋CSSだけでは、ボタンの見た目は作れても動きは作れません。
見た目と動きは別の技術が担当しています。ボタンに「押したら何かする」という命令を与えるのがJavaScriptの役割です。
Q2
問題: ボタンを押すとメニューが開き、入力すると候補が出る。
この「操作に反応する動き」を実現しているのはどれ？
正解: A. JavaScript
解説: HTMLは構造、CSSは見た目を担当します。「操作に反応して画面が変わる」動きを作っているのがJavaScriptです。
操作に反応して内容が変わる動きの多くは、JavaScriptが担っています。
Q3
問題: HTML＋CSSだけのページにJavaScriptを追加したら、どう変わったでしょう？
正解: A. ボタンを押すとメニューが展開されるようになった
解説: JavaScriptを追加すると、ボタンを押したときにメニューが展開されるようになりました。
HTML＋CSSだけでは、操作に応じて内容を書き換えることはできません。JavaScriptが加わると「操作に反応する画面」に変わります。
Q4（マッチング）
問題: 「家を建てる」で考えましょう。Webページの3つの技術と、家づくりの役割を正しく対応させてください。
正解:
HTML ↔ 骨組み
CSS ↔ 内装
JavaScript ↔ 家電
解説: HTMLは骨組み、CSSは内装、JavaScriptはスイッチを押すと動く家電です。
家電があると「住人の操作に反応する家」になります。JavaScriptがWebページに動きを与えます。
Q5（穴埋め）
問題: JavaScriptが加える「動き」を、正式にはどう表現する？
正解: 動的な振る舞い
解説: JavaScriptはWebページに動的な振る舞い（操作への反応、表示の変化など）を追加する言語です。
構造はHTMLの仕事、見た目はCSSの仕事。JavaScriptの仕事は「動き」です。
Q6
問題: JavaScriptが画面を書き換えるとき、ブラウザがHTMLから組み立てた部品の集まりを操作します。
この部品の集まりを何と呼ぶ？
正解: A. DOM
解説: DOMは、ブラウザがHTMLを読んで組み立てるページ構造です。JSがこの構造を書き換えると画面も変わります。
DOMの仕組みは、この先で詳しく見ていきます。
Q7
問題: 3つの技術が層になっています。図で一番上に位置するJavaScriptは、どんな役割を担っていますか？
正解: A. ユーザーの操作に反応して画面を変化させる
解説: JavaScriptは操作に反応して画面を変化させる役割です。HTMLが構造を、CSSが見た目を整えた上に、JavaScriptが「動き」を加えます。
3つの技術が揃うと、ユーザーの操作に反応する、より豊かなWebページが完成します。
Q8（◯×）
問題: 「静的」なページは入力しても表示が変わらない。「動的」なページでは入力に応じて変わる。
ブラウザ上でこの違いを生んでいるのはJavaScriptである。
正解: 正しい
解説: HTML＋CSSだけでは、ページの表示内容は固定されたまま変わりません（静的ページ）。
JavaScriptを加えることで、入力に応じて表示が変わる動的ページになります。
Q9
問題: JavaScriptの仕組みを知ると、何ができるようになりますか？
正解: A. Webサイトの動きが何で実現されているかを推測できる
解説: JavaScriptの仕組みを知ると、「ボタンを押すとメニューが開く仕組みはこうだったのか」と裏側が推測できる目を手に入れられます。
ブラウザの開発者ツール（Consoleタブ）で、JavaScriptが裏で何をしているか覗くこともできます。
Q10（穴埋め）
問題: 家を建てるメタファーで、それぞれの技術はどの役割？
正解: HTML / CSS / JavaScript
解説: 骨組み（構造）はHTML、内装（見た目）はCSS、家電（動き）はJavaScriptの担当です。
家電が加わることで、見た目だけの家が「操作に反応する住める家」に変わります。
Lesson 2: ブラウザが組み立てる構造——DOMの仕組み
Q1
問題: JavaScriptで画面の文字を書き換えたとき、HTMLのソースコード（設計図）はどうなっているでしょう？
正解: A. ソースコードは元のまま変わっていない
解説: 実は、JSが画面を書き換えてもHTMLのソースコード自体は変わりません。JSが操作しているのは、ソースコードとは別のものです。
では、JSは一体「何を」書き換えているのでしょう？ その正体が今回のテーマです。
Q2
問題: ブラウザはHTMLを読み込んだ後、画面を表示する前に何かを組み立てています。
その「何か」に最も近いのはどれ？
正解: A. HTMLの内容を元にした操作可能な構造
解説: ブラウザはHTMLを読むと、その内容を元に操作可能な構造を組み立てます。
JSが画面を書き換えられるのは、この構造を操作しているから。次の問題で、この構造のイメージを掴みましょう。
Q3
問題: ブラウザがHTMLから組み立てたページ構造（DOM）。JavaScriptがこの構造の一部を書き換えるとどうなる？
正解: A. 画面の表示もすぐ変わる
解説: ブラウザが組み立てたページ構造（DOM）を書き換えれば、画面もすぐ変わります。
JavaScriptは、この構造の部品を直接操作することで画面を書き換えているのです。
Q4
問題: ブラウザが組み立てるページ構造（DOM）は、いつ作られるでしょう？
正解: A. ブラウザがHTMLを読み込んだ後、自動的に作られる
解説: DOMはHTMLファイルに最初から入っているわけではありません。ブラウザがHTMLを読み込んだ後に、自動的に構築されます。
ブラウザが設計図（HTML）を読んで、ページ構造を組み立てる——そんなイメージです。
Q5（穴埋め）
問題: ブラウザがHTMLから組み立てる「操作可能なページ構造」の正式名称は？
正解: DOM
解説: 正式名称はDOM（Document Object Model）。「ページの中身を、操作できる形に組み立てたもの」という意味です。
ブラウザがHTMLを読んで組み立てたページ構造——それがDOMです。
Q6
問題: DOMは「ツリー（木構造）」の形をしています。上が親、下が子です。
bodyから直接ぶら下がっている（＝bodyの子）要素はどれ？
正解: A. h1 と p
解説: ツリーを見ると、bodyから直接枝が伸びているのはh1とpです。headとbodyはhtmlの子であって、bodyの子ではありません。
この「親→子」の関係を読み取れるのがDOMツリーの力です。
Q7
問題: HTMLコードとブラウザ表示を見比べてください。大きく太い文字で表示されているのはどのタグの内容？
正解: A. <h1>タグの内容
解説: 「ようこそ」が大きく太い文字で表示されているのは、<h1>タグで囲まれた内容だからです。
ブラウザはHTMLを読み込んでDOMを作り、DOMの各要素に対して画面を描画します。JSで画面を変えたいとき、操作する相手はこのDOMの中の要素です。
Q8（◯×）
問題: JSがDOMを書き換えると、画面の表示もリアルタイムで変わる。
正解: 正しい
解説: DOMは画面と連動しています。JSがDOMの要素を書き換えると、画面もすぐに変化します。
これがJSで画面を動かせる仕組みの正体です。ページ構造（DOM）を書き換えれば、画面が変わるのです。
Q9
問題: ブラウザには「開発者ツール」という裏側を覗ける機能があります。その中のElementsパネルを開くと表示されるのは？
正解: A. ブラウザが構築したDOMツリー
解説: Elementsパネルに表示されるのはDOMツリーです。HTMLソースコード（設計図）の写しではなく、ブラウザが組み立てたページ構造（DOM）を見ています。
JSでDOMを書き換えると、Elementsパネルの表示もリアルタイムで変わります。
Q10
問題: DOMで画面を書き換えられる。では、ユーザーがボタンを押したことをJSはどうやって知る？
正解: A. DOMの要素に「検知する仕組み」を取り付けておく
解説: DOMの要素に「検知する仕組み」をあらかじめ取り付けておくと、クリックや入力が起きたとき、JSに自動で通知が届きます。
取り付けは開発者の仕事、検知と通知は自動——この仕組みの正体と名前に、この先で迫ります。

このコースのキーポイント
JavaScriptは「動き」を担当——HTMLとCSSだけでは「読むだけの紙」だったページが、「触ると反応する家」になる。
DOM（Document Object Model）: ブラウザがHTMLを読んで組み立てる「ツリー状のページ構造」。JSはこのDOMを書き換えることで画面を変える。HTMLソースコード自体は変わらない。
イベントとイベントリスナー: 「クリックされた」「キーが押された」「スクロールされた」などのできごとを「イベント」と呼ぶ。要素にセンサー（イベントリスナー）を付けると、イベントが起きたときにJSに通知が入る。
主なイベント名: 
  click → クリック・タップ
  input → 入力欄への文字入力
  scroll → 画面スクロール
  submit → フォーム送信
静的 vs 動的: JSなしのページは静的（表示が固定）、JSありのページは動的（操作に反応して内容が変わる）。SNSの「いいね」が押したときに数字が増えるのも、検索ボックスでリアルタイムに候補が出るのも、すべてJSの動的ページの例。
フロントエンド vs バックエンド: HTML/CSS/JSで「ブラウザの中で動く部分」を作るのがフロントエンド、サーバー側で動くのがバックエンド。
JavaScriptはブラウザだけではない: Node.jsでサーバーでも動く、React Nativeでスマホアプリも作れる。一つの言語でもう両方でも動く汎用性の高さがJSの強み。
代表的なJSコードパターン
// 要素を取得
const btn = document.querySelector('button');

// イベントリスナーを付ける
btn.onclick = () => {
  // DOMを書き換える
  const msg = document.querySelector('#msg');
  msg.textContent = '更新しました！';
  msg.style.color = 'green';
};
「要素を取る」「イベントを付ける」「DOMを操作する」という3ステップがフロントエンドJSの基本パターンです。
