4月以降、特進コースの生徒の割り振り方法を変更したいと思います。
【背景】
・生徒の入会から30日以内に1投稿目が完了できるよう、SPの内容が50日→30日で完了するようになる
・SPの初期設計のタイミングがSTEP3にあるため、今まで通りアサインまでに1週間以上時間を要すると、このスピード感に対応できない
・これまでのアサインの方法は、生徒の入会人数が3名や2名/月だった時代のやり方をそのまま引き継いでおり、入会人数が毎月20名以上、なんならこれから毎月40名〜50名を受け入れられる状況を作ると考えた時、現行の進め方だと物理的に運営破綻

## 生徒のフロー
入会
└─→ スターター（step0 特進コースについての説明動画視聴・自己分析セッションの予約完了）
　-→アサイン完了後、MGのスレッドへ案内される
　-→（18STEP / 10日）
      └─→ 通常運用（以下のサイクルを繰り返す）

『30日ごとの「期間」サイクル（7期間 = 210日）』
期間開始 ─→ MGとの自己分析セッション
   │    
   ├─→ ◆自己分析セッション
   │    ・自己分析シートを埋める
   │    
   ├─→ ◆2回目セッション（MG顔合わせ・目標設定セッション）
   │    ・数値KPI設定（収益/投稿数/フォロワー数）
   │    ・1週間のタスク決定（stepいくつまでSP完了させるか）
   │    ・週次タスクシート　埋める 
   │    ・1投稿目作成完了日（添削完了したもの、いつでも投稿できる状態のものを完成させる）
   │
   ├─→ ◆セッション（1ヶ月目：週1回、2,3,4ヶ月目：2週回、5,6ヶ月目：週1回/. 合計12回）
   │    ① 前回のセッションからの振り返り（1週間、2週間、1ヶ月の実績）
   │    ② 次2週間の「週次タスクシート」作成
   │       → 投稿目標本数やその他アクションを設定
   │
   ├─→ 日常行動（週次タスクシートに従う）
   │    ・今日の時間ブロックを確認 → タスクを完了
   │    ・投稿ワークフロー（10工程/1投稿）
   │    ・定期タスク（ラジオ/エスキャン/週報等）
   │
   └─→ 期間終了 → 振り返り（月初1〜10日でMGと一緒）
        ・投稿状況の「結果/要因/改善案」
        ・その他の「結果/要因/改善案」
        ・数値KPI結果を入力
## MGのフロー
日常
 ├─→ 目標設定シートで決めたタスクについて、担当生徒の進捗確認・遅れアラート
 │    ・自己紹介
 │    ・次回セッションまでのタスクを、時間帯と一緒に約束
 │    ・2回目通常セッション 日報の提出時間を確定
 │    ・誓約書
 └─→ セッション時（生徒と対面 / PC使用）
      ① 前回2週間のタスクシート実績確認
      ② 投稿目標本数やその他アクションを設定
      ④ MGメモ（自分用）+ 生徒向けメッセージ入力
      ④ 期間終了時: 生徒と一緒にKPI結果入力・振り返り記入支援

リーダー
 ├─→ 配下MGの担当生徒を横断閲覧
 ├─→ 進捗遅延の生徒にアラート確認
 └─→ チーム全体のKPIサマリー確認
