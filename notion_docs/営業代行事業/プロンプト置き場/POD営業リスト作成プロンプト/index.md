## 目的
出版経験のある個人クリニック（開業医経営）の営業リストを、重複を徹底的に排除した形で作成する。
## 要件
**対象条件（必須）**
1.個人クリニックであること
- 全国展開している大規模な医療法人は除外
- チェーン展開しているクリニックは除外
- 開業医が自らそのクリニックを経営・診療しているケースに限定
- 病院勤務医や大学病院勤務のみの医師は除外
2.出版経験があること
- 代表者（院長）が書籍の出版経験があること
- 一般向けの健康・医療関連書籍が対象
- 学術論文のみは対象外
3.重複排除（最重要）
- 既存の営業リスト（Googleスプレッドシート、CSV）と重複しないこと
- 企業名（クリニック名）での重複チェックを徹底
- 院長名での重複チェックも実施
- 同一クリニックの別院・分院も重複として扱う

## 重複チェックの徹底（最重要）
**既存リストとの照合方法**
1.既存リストの取得
- Googleスプレッドシート: https://docs.google.com/spreadsheets/d/13R6v-y2pxMGlt_roSZDVop-ZbdMnFBkKvmYoAKQVJOg/edit?gid=1768735528#gid=1768735528
- ローカルCSVファイル: clinic_list_200.csv、final_clinic_list.csv
2.重複チェックの実施タイミング
- 調査前: 既存リストを完全に読み込み、メモリに保持
- 調査中: 候補を見つけるたびに既存リストと照合
- 調査後: 最終リスト作成前に再度全件チェック
3.重複判定の基準
- 企業名（クリニック名 ）が完全一致: 重複として除外
- 企業名が部分一致: 慎重に確認（例: 「〇〇クリニック」と「医療法人〇〇 〇〇クリニック」）
- 院長名が一致: 同一人物の可能性が高いため、クリニック名も確認
- 分院・別院: 同一医療法人の別拠点は重複として扱う
4.重複チェックのコード例（Python）
Python
import csv

# 既存リストを読み込み
existing_clinics = set()
existing_directors = set()

with open('existing_list.csv', 'r', encoding='utf-8') as f:
    reader = csv.DictReader(f)
    for row in reader:
        existing_clinics.add(row['企業名'].strip())
        existing_directors.add(row['代表者（院長）'].strip())

# 新規候補のチェック
def is_duplicate(clinic_name, director_name):
    # 完全一致チェック
    if clinic_name in existing_clinics:
        return True, f"企業名が重複: {clinic_name}"
    
    # 院長名チェック
    if director_name in existing_directors:
        return True, f"院長名が重複: {director_name}"
    
    # 部分一致チェック（クリニック名の主要部分）
    for existing in existing_clinics:
        # 医療法人名を除いた部分での比較
        clinic_core = clinic_name.replace('医療法人', '').replace('社団', '').replace('財団', '').strip()
        existing_core = existing.replace('医療法人', '').replace('社団', '').replace('財団', '').strip()
        
        if clinic_core in existing_core or existing_core in clinic_core:
            return True, f"企業名が部分一致: {clinic_name} ≈ {existing}"
    
    return False, "重複なし"

# 使用例
is_dup, reason = is_duplicate("銀座よしえクリニック", "青木 晃")
if is_dup:
    print(f"除外: {reason}")
## 注意事項（過去の失敗から学んだポイント）
**❌ 避けるべき失敗パターン**
1.重複チェックが不十分
- 問題: 既存リストとの照合を怠り、同じクリニックを何度も調査
- 対策: 調査開始前に既存リスト全件をメモリに読み込み、常に参照
2.有名医師の重複調査
- 問題: 南雲吉則、和田秀樹、近藤誠など有名医師が繰り返し候補に挙がる
- 対策: 有名医師リストを事前に作成し、優先的に除外
3.大学病院勤務医を含めてしまう
- 問題: 個人クリニックを経営していない医師を含める
- 対策: クリニックの公式サイトで「院長」として勤務していることを確認
4.故人を含めてしまう
- 問題: 日野原重明など故人の医師を含める
- 対策: 最新情報で生存確認、またはクリニックの現在の運営状況を確認
5.電話番号の確認不足
- 問題: 古い情報や誤った電話番号を記載
- 対策: 公式サイトまたは医療情報ネット（厚生労働省）で最新情報を確認
6.並列調査での重複増加
- 問題: 100件の並列調査で同じ候補が複数回調査される
- 対策: 並列調査の候補リスト作成時に、既に調査済みの医師を除外
## 効率的な調査戦略
**1. 段階的なアプローチ**
- 第1段階: 有効率の高い情報源から調査（オーソモレキュラー医学会など）
- 第2段階: Amazon書籍データベースから医師著者を抽出
- 第3段階: 地域別・診療科別の網羅的調査
**2. 並列処理の活用**
- 一度に50-100件の候補を並列調査
- ただし、候補リスト作成時に既存リストとの重複を事前チェック
**3. 有効率の目標**
- 第1-3次調査: 有効率30-40%を目標
- 第4次以降: 有効率20%程度に低下するが継続
- 有効率が10%を下回ったら情報源を変更

## 調査の進め方（推奨手順）
**ステップ1: 準備**
1.既存の営業リストを全て取得（Googleスプレッドシート、CSV）
2.既存リストをメモリに読み込み、重複チェック用のセットを作成
3.有名医師の除外リストを作成
**ステップ2: 候補の抽出**
1.情報源から候補医師をリストアップ（100-200件）
2.候補リストと既存リストを照合し、重複を除外
3.残った候補のみを並列調査の対象とする
**ステップ3: 並列調査**
1.50-100件ずつ並列調査を実施
2.クリニック名、電話番号、著書を確認
3.個人クリニック経営かどうかを確認
**ステップ4: 結果の精査**
1.並列調査の結果から有効な候補を抽出
2.再度、既存リストとの重複チェック
3.電話番号と著書の情報を最終確認
**ステップ5: リスト作成**
1.有効な候補のみをCSV形式で出力
2.品質保証チェックリストで確認
3.100件ごとにユーザーに提供
## トラブルシューティング
**問題: 重複が多発する**
原因: 既存リストとの照合が不十分 対策: 調査前に既存リスト全件を読み込み、候補抽出時に事前チェック
**問題: 有効率が低い（10%以下）**
原因: 情報源が枯渇、または大学病院勤務医が多い 対策: 情報源を変更（地域別、診療科別など）
**問題: 電話番号が見つからない**
原因: クリニックが閉院、または情報が古い 対策: 医療情報ネット（厚生労働省）で確認、見つからない場合は除外
**問題: 著書が確認できない**
原因: 書籍情報が古い、または絶版 対策: Amazon、国会図書館サーチ、出版社サイトで確認
