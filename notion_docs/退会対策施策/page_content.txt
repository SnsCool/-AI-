# 退会対策施策

  対象期間：2026年5月9日〜6月26日回答件数：21件（うち1名重複回答のため実質20名）／全員ライトコース
  エグゼクティブサマリー
  退会者**20名のうち半数（10名）が「満足／とても満足」と回答しており、退会の主因は講座の質への不満ではなく、「使いこなせなかった」「本人の状況・方向性が変わった」**ことにある。
これは裏を返せば、仕組みの改善で防げる退会が多いことを意味する。特に「夜のリアルタイム（グルコン）への参加依存」が、時間制約のある会員（主婦・本業持ち）の脱落を招いている点が最大の構造課題である。
  1. 退会理由の分類（20名）
  2. 満足度の分布
  重要な発見：退会者の50%が「満足／とても満足」のまま退会している。
退会＝サービス不満ではなく、「立ち上がれなかった」「状況が変わった」が本質。打ち手で防げる余地が大きい。
  3. 自由記述から見える3つの構造的問題
  ① リアルタイム依存 → 参加できない人が「相談できない＝価値ゼロ」になる
  最も繰り返し現れたパターン。
  こみね（満足）「環境変化で学習時間が確保できない」
  ざんま「本業＋子の寝かしつけで21時のグルコンに参加できない」
  きし「リアルタイムzoomに参加しないと相談しづらい。希望講師のグルコン回数が少ない」
  くろさわ「グルコンに参加できずサービスを活用できていなかった」
  くもん「リアルタイム講座も大人数で質問しきれなかった」
  → 「夜のリアルタイム・グルコン」が価値の中心になっており、参加できない層が脱落している。
  ② 個別・非公開で相談できる場の不在
  こみね「マネタイズの訴求はグルコンで聞けるが、自分のアカウントの方向性を、周りに知られず本音で話せる場所がなかった。入会時1on1はノウハウ前で活かせず、やってみた後に相談する機会が欲しかった」
  → 1on1が「入会直後の早すぎるタイミング」に固定されている。
  ③ プラットフォーム/方向性のミスマッチ（満足だが離脱）
  いけだ（とても満足）「TTS/TikTokに移行。Instagram中心で合わず。TTSの学びをリクエストしたが…」
  くきの（とても満足）「Instagram止めてTikTok Shopへ移行」
  こじま「子育て系アカウントは内容が合わないことが多い」
  → 満足度が高いのに、対象プラットフォームが本人の事業と合わなくなって離脱。
  4. 「防げた退会」と「防げない退会」の切り分け
  防ぎにくい（自然減・約4名）
新しい仕事が決まった（えの）、他を優先（ふるしょう）、環境変化の一部。
  防げた可能性が高い（過半数）
  時間制約組（4〜5名・満足度高め）→ 非同期で完結できれば残れた
  プラットフォーム移行組（いけだ・くきの・とても満足）→ TikTok導線があれば残れた（いけだは自ら要望していた＝最も惜しい離脱）
  料金・プラン組（すずき・やすたけ・たつみ）→ たつみは「エリートを検討したが会費＋収益30%がネックで踏み出せなかった」＝中間プランがあれば上位転換できた
  5. 対策案（優先度順）
  【最優先】夜のリアルタイム依存からの脱却
  グルコンの時間帯分散（昼／朝枠の新設）＋録画前提の設計
  非同期の質問チャネル（チャットで個別質問 → 講師が回答、テキストでも完結）。「参加できない＝相談できない」を解消する。
  【最優先】1on1のタイミング再設計
  入会直後1回固定をやめ、「実践してみた後（1〜2ヶ月後）」にもう1回を組み込む。
  【高】非アクティブ会員への早期介入（オンボーディング）
  「どちらともいえない」8名の多くは**「全く見ていない」「活用できていなかった」＝そもそも立ち上がっていない**。
  最初の30／60日で未着手者を検知して声かけする仕組み（ログイン・視聴ゼロのアラート）。
  【高】退会前の引き止め選択肢（ダウンセル）
  全退会の前に「休会」「下位／低価格プランへの変更」を提示。
  たつみ向けに**「エリート未満・1on1付きの中間プラン」**を検討（収益30%シェアが心理的障壁）。
  【中】TikTok／他プラットフォーム導線
  TikTok Shop・TTSへの移行需要が複数。簡易トラックやアーカイブを用意すれば、満足度の高い層を引き留められる。
  6. データ運用上の指摘
  退会率を正確に算出するには現在のアクティブ会員数が必要。
（過去分析では3〜6月で48件のライト課金 → これに対し退会20名は churn が高めの可能性。母数を確認の上で退会率を算出すべき。）
  次のアクション候補
  入会ファネル（AW通知 → LINE → 1on1 → 入会）と突き合わせ、どの段階で入った人が辞めやすいかを分析する。
  上記対策のうち、まず着手するものを運営で選定する。
