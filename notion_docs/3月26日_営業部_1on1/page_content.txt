# 営業部の雰囲気改善と営業プロセス見直し
山口有紗によると、営業部の雰囲気が良くなってきており、これは一見さんの功績が大きいと考えている。一見さんとのミーティングの結果、マーケティング関連の営業担当者は一旦全員で担当し、制約率などのデータを見て得意な人が出てきたら担当を絞る方針になった。
# 新しい評価制度の導入検討
現在の営業プロセスには不正防止のためのルールが多く、管理が難しく、メンバーの委縮にも繋がっているという課題がある。これに対し、一見さんが過去の組織で採用していた評価制度を導入したいと考えており、面談数、着座率、制約率のそれぞれにランク（例：S, A, B）をつけ、最も低いランクに合わせて評価と報酬を決める制度を検討している。これにより、報酬を下げないために全てのプロセスを適切に実施する必要がある状況を作る。
# コンプライアンスチェックの基準組み込みとチーム再編
コンプライアンスチェックについて、現在は機能していないため、新評価制度の中に組み込むことを松本実緒子に依頼する予定である。また、チームの入れ替えを計画しており、明日中には完了させる意向である。
# マネージャー育成と組織の将来的な方向性
今後の組織が目指す方向性として、純夜とレオがマネージャーの輩出に注力する動きが共有された。純夜は武蔵と司、レオはひとよしを育成し、最終的には4つのチームに分ける構想である。長期的には、山口有紗が多くのマネージャーを管理しきれなくなるため、レオと純夜を「マネージャーのマネージャー」として配置し、営業部を統括する階層構造を構築したい意向である。
# マネージャーのモチベーション維持とインセンティブ
マネージャーがモチベーションを高く維持するためには、育成したメンバーがチームから離れた際にプラスのインセンティブを付与することを検討している。役職を排出した場合のインセンティブがあれば、トッププレイヤーを育てたにもかかわらずチームを離れることで報酬ベースでの育成の意味が薄れるという問題の解決に繋がると考えられる。
# マネタイズ講座に関する問題点の特定
幹部チャットで共有されたマネタイズ講座に関する動画を全体的に見た結果、問題は講師とマネタイズ講座の営業にあると結論付けられた。特に、営業担当者の教育が不十分で態度が悪いこと、また担当講師がフォロワーを伸ばすことに特化しておりマネタイズをあまりやっていないスタンスであった可能性が指摘された。
# チーム再編と新制度導入の慎重な進め方
チームの再編成と新しい評価制度の導入は、過去に不満を感じたメンバーがいた経緯があるため、慎重に進める必要があるとの懸念が示された。新しい制度は、現状の複雑なルールを整理し、分かりにくい状況を解消するという見せ方で進めることが提案された。
# チームリーダー間の連携に関する懸念
幹部会議で導入が決定した指標に関して、データ入力が追い付いておらず機能不全の状態になっていることが確認された。また、面談数などの要素によって制約率の難易度が異なるため、単純にこの指標だけでは判断しきれないという現状も共有された。
# コンプライアンスに関する再調整の必要性
評価制度は不正をなくす方向に持って行くため、各要素にランク付けをするという形で再調整を進めることになった。コンプライアンスに関しては、完全に点数化するのは難しいため、AIでアラートが鳴った際に目視で確認し指導する形が良いとの見解が示された。松本実緒子から、以前から松本実緒子自身が個別で見かけていたコンプラ違反に関する注意喚起の重要性が共有され、今後はサポートマネージャーのボムさんやCS側との定期的な連携が必要であると判断された。
# 熱量のある発言の適切な場に関する議論
レナさんが全体チャットで行った熱意ある呼びかけについて、組織として動く中で、それはマネージャーや統括など上の立場の人間が言うべきであるという松本実緒子の意見が共有された。特に、レナさんがお手本となるほどの制約率ではないため、周りのメンバーの人間関係に軋轢を生じさせかねないという懸念が示された。
# 営業マネージャー層への情報共有の検討
現在、マネージャーミーティングや一見さんとのミーティングの情報をどこまで共有するかについて、マネージャー層が幹部の一つ下のレイヤーであるため、アイデアやタスクを一旦別の部屋で共有した上で、山口有紗や大さんが最終判断を行う形が提案された。
# 愚からの評価と感謝
愚から山口有紗へ、皆で旅行を計画したいという発言に対しての感謝のメッセージがあり、山口有紗は愚や純夜が自身の考えを信頼してくれていることに喜びを感じている。
# タスク管理ツールの整理と運用方法
現在のタスク管理が煩雑になっており、メインタスクがサブタスクに適切に紐づいていない問題が確認された。解決策として、組織課題など大きなカテゴリー（親タスク）を用意し、その下に具体的な実行事項をサブタスクとして追加していく方法が提案され、今日中に整理を進めることになった。
# マネージャーのタスク可視化の重要性
レオなどのマネージャーが担当するタスクについて、具体的な行動内容と期限をサブタスクとして記載することで、レオ任せにせず、幹部間で状況を把握できるようにする重要性が再認識された。
# キャンセル枠管理の担当と目的
キャンセル枠の管理を営業側にもやってもらう件について、現在多忙なシュリの負担軽減のため、また、デビュー前のベンチメンバーを枠に入れるかどうかを判断する最適なタイミングを有川さんが把握しているため、有川さん、レナさん、チサの4人体制で協力して行うことが提案された。
# リアルタイムでの制約率把握の問題と解決の方向性
メンバーが自身のリアルタイムの制約率を把握できていない問題があり、純夜と一見さんが改善に取り組むことになった。このデータはチームごとや、シュリ、有川さんなど各担当者ごとの軸で出すことで、原因の特定と改善策の明確化に繋がるとされた。
# 成功事例の要因特定と個別の施策展開
成功事例（例：司の施策）があった場合、その施策自体が良いのか、それとも実施者のマインドやタイプに合っていたから成功したのかを特定することが重要であるという認識が共有された。今後は、個人に合わせた施策を実施することが有効であるとの見解が示された。
# 長期的な施策の必要性
過去の施策（例：ナさんのスクリプト化）では、一部のメンバーにしか効果がなかったため、大人数の組織全体に劇的な改善をもたらすためには、一見さんとシュリが主導し、タイプ別のトークスクリプトを作成・展開するという長期的な施策が必要であると結論付けられた。この施策は4月のローンチまでに完了させる予定である。
# チーム間の数字共有における懸念
オロチームがチーム内チャットで共有している制約率が、確定分ではなく予定ベースの数字で計算されている可能性が指摘された。これはチームの雰囲気を上げるための意図的な行動である可能性があるが、現実的な数字で目標意識を持たせるため、一見さんを通じてオロに意図を確認し、着座ベースの数字を意識させるように指導することが提案された。
# CT室へのメンバー追加
田中さんと吉田さんをCT室に加えることについて、松本実緒子は承認した。有川さんについては、教育担当としてリソースがあること、またCTの仕事を楽しんでいるため、両方継続して担当してもらい、4名体制で進めることが最終決定された。
# Googleアカウントの移行に関するタスク
山口有紗がGoogle Meetのリンクを適切に利用できるようにするため、松本実緒子と山口有紗のGoogleアカウントを移行する必要があることが確認され、松本実緒子が具体的な手順を再度確認して山口有紗に伝えることになった。
# 報酬制度と収入の懸念
山口有紗は、固定報酬がなくなり、結果が出なければボスも渡せないため、二人とも結果がなかった場合の収入の不安について懸念を表明しました。参加者たちは、結果を出すしかないという姿勢であったため、山口有紗は本人たちが良ければ良いが、責任は取れないという考えを示しました。
# タスクの処理と整理
山口有紗は、予定されていた数字に関する混乱と、事前キャンセルが多すぎるという問題（特にレナちゃんのキャンセルが多いこと）に触れ、数字の確認が必要であると述べました。処理塾に人が入っているものの、スクしても状況がわからないため、タスクの整理を行うことが決定されました。このタスクの整理は、小さなチームに投げるのは厳しいため、人事のタスク整備ができていなかった人が厳しく追及されていたことを踏まえ、来週囚人渚さんが整理をし、月曜日頃に山口有紗が確認する予定となりました。
