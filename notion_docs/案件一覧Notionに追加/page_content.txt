Notion案件管理データベース作成依頼
Instagramマネタイズ講座の生徒向けに、
アフィリエイト・固定案件の情報を一元管理するNotionデータベースの作成をお願いします。
目的は、
生徒が「どの案件を選べば良いか」を一目で判断できる状態にすることです。
単なる案件リストではなく、成果が出やすい案件・報酬が高い案件を比較しやすい設計にしてください。
まずは箱を作っていただき、そこから各講師や生徒が入力していくものとします。とにかく見やすい、探しやすいを意識して構築いただきたいです。ポータルの中に格納するイメージです。
1. データベース構造
Notionの**フルページデータベース（Table）**で作成
1案件＝1行で管理
生徒が目的別に探しやすいよう複数ビューを設定
2. 必須プロパティ（項目）
以下のプロパティを設定してください。
案件名
タイトル
収益化おすすめ度
セレクト
★★★★★ / ★★★★☆ / ★★★☆☆ / ★★☆☆☆ / ★☆☆☆☆
案件タイプ
セレクト
アフィリエイト / 固定案件 / PR案件
ジャンル
マルチセレクト
例：美容 / ダイエット / 副業 / 恋愛 / 金融 / 転職 / ライフスタイル
報酬額
数値またはテキスト
（1件あたり報酬・固定報酬FW数×いくらなど）

テキスト
例：購入 / 無料登録 / LINE追加 / 面談予約
フォロワー相性
テキスト

ASP / 依頼元

3. 作成してほしいビュー
生徒が案件を探しやすいよう、以下のビューを作成してください。
① 総合おすすめ順（デフォルト）
Table
並び替え：総合おすすめ度（降順）
② ジャンル別一覧
Board
グループ化：ジャンル
③ 初心者向け案件
Table
フィルター：初心者向きか（チェックあり）
④ 高単価案件一覧
Table
並び替え：報酬額（降順）
⑤ 固定案件・PR案件
Table
フィルター：案件タイプ＝固定案件 / PR案件
ページ上部のガイド
データベースの上部に、生徒向けの簡単な使い方ガイドを
Calloutブロックで配置してください。
記載内容（例）
このデータベースの使い方
・まずは「総合おすすめ順」または「初心者向け案件」から、自分に合う案件を探してください。
・おすすめ度は「報酬の高さ」と「成果の出やすさ」を基準に評価しています。
・各案件ページを開くと、推奨訴求例や注意点を確認できます。
その他
・セレクトタグは視覚的に分かりやすい色分けを設定してください
（例：おすすめ度が高い案件は目立つ色）
・各案件ページ内には、
訴求テンプレート・参考投稿・メモ等を追記できるフリースペースを残してください。
