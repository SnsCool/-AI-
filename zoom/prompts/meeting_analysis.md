# 商談分析プロンプト

## 目的
Zoom面談の文字起こしから、営業の良し悪しを分析し、成功ナレッジを蓄積する

## 分析項目

### 1. クロージング判定
| 判定 | 条件 |
|-----|------|
| 成約 | 契約・申込み・購入が確定した |
| 未成約 | 明確に断られた、検討しないと言われた |
| 継続 | 次回アポ設定、検討中、資料送付後に連絡 |

### 2. 話す割合
- 理想: 営業30-40% / 顧客60-70%
- 顧客が多く話しているほど良い傾向

### 3. ヒアリングした課題
- 具体的な困りごと
- 数字で表現された課題
- 感情を伴う課題（困っている、イライラなど）

### 4. 提案内容
- 課題に対する解決策
- 具体的な機能・サービス
- 導入後のイメージ

### 5. 良かった点（評価基準）
- 課題を深掘りできた
- 数字で効果を伝えた
- 具体的な日程を提案した
- 顧客の言葉を繰り返した（傾聴）
- 沈黙を恐れなかった
- 反論に適切に対応した

### 6. 改善点（評価基準）
- 話しすぎた
- 質問が少なかった
- 提案が課題と紐づいていない
- クロージングが弱い
- 専門用語を使いすぎた
- 価格の説明が曖昧

### 7. 成功キーワード
成約・継続商談から抽出するパターン:
- ヒアリング手法（例: 課題の深掘り、数字での確認）
- 提案手法（例: 導入事例の提示、ROI説明）
- クロージング手法（例: 具体的日程提案、限定感の演出）

## 出力形式

```json
{
  "closing_result": "成約 | 未成約 | 継続",
  "talk_ratio": {"sales": 40, "customer": 60},
  "issues_heard": ["課題1", "課題2"],
  "proposal": ["提案1", "提案2"],
  "good_points": ["良かった点1", "良かった点2", "良かった点3"],
  "improvement_points": ["改善点1", "改善点2", "改善点3"],
  "success_keywords": ["キーワード1", "キーワード2"],
  "summary": "商談の要約（3-5文）"
}
```
