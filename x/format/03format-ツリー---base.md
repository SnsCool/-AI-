# ツリー（スレッド）基本フォーマット

## フォーマット概要
複数のツイートで構成されるスレッド投稿の基本フォーマットです。詳しい内容を段階的に説明します。

## 構造
```
[1/5] [フック] - スレッド全体のテーマ
[2/5] [背景] - 問題や状況の説明
[3/5] [詳細1] - 具体的な内容や事例
[4/5] [詳細2] - 追加の説明や実例
[5/5] [まとめ] - 結論と行動喚起
```

## 使用例
```
[1/5] 🧵 今日は、LaravelでAPIを設計する際のベストプラクティスを5つ紹介します

[2/5] 多くの開発者が見落としがちなポイントがあります。適切な設計をしないと、後から大幅な修正が必要になります

[3/5] 1. RESTfulな設計を心がける
   - リソースベースのURL設計
   - 適切なHTTPメソッドの使用
   - ステータスコードの統一

[4/5] 2. バリデーションの実装
   - リクエストデータの検証
   - エラーメッセージの統一
   - セキュリティの考慮

[5/5] これらのポイントを意識することで、保守性の高いAPIが作れます。他にも知りたいポイントがあれば、コメントで教えてください！ #Laravel #API #設計
```

## 効果的な要素
- 各ツイートに番号を付ける
- 絵文字で視覚的に区別
- 段階的な情報の提供
- 最後にインタラクションを促す
